Common Cores

Computers Architecture and networking

The objective of this course is to provide to the students the comprehension of the fundamental mechanism of the Operating Systems and the concurrence on one side, and of the networks based on the TCP/IP protocols on other.

Concerning the Operating Systems: we have detailed on the the processes and memory management, several systems calls by using C language. The basic services of POSIX norm were also presented. This part was completed by a presentation on the techniques of virtual machine.

Concerning the networks: we have learned several topic about networks and transport layers. There were also presentations on a few protocols following the TPC/IP that would be illustrated by a public domain tool and also a detailed study on the socket programming interface.

Program:

  • Concurrent and parallel programming
  • OS, processors and memory architecture
  • Networks
  • Concurrent programming in practice: POSIX
  • Concurrent program design pattern.
Tag: C, shell, ns-2, GNUPLOT, wireshark

Computers and Networks Security

The informatique security aims to protect the computers and information systems against the malicious attacks. These issues comprise many technical aspects that could be down to the medium level, to software, to the environment and as well as to the social behaviors of the users. The main goal of this course is to treat the fundamental principals of informatique security.

Program:

  • Introduction to the informatique security and its fundamental concepts.
  • The necessity of informatique security and its usage.
  • Security organisation in organisations.
  • Risk analysis and its concrete usage in organisations. (This course was not held due to covid-19).
Tag: Cryptography, OSI attacks, confidentiality, signatures, python API (hashlib, scrapy)

Computer Science Track

Real-Time Systems

In this course, we are interested in the real-time systems that are present in various applications such as Aeronautics, space, nuclear power plants, train or even medical. The proper functionning of these systems is essential to meet user requirements. This fact depends not only on the accuracy of the result povided, but also on the time of their generation. In this course we study the performance of different systems architectures of the mono-processor, multiprocessor and communications systems via on-board networks. This study take into account the impact of scheduling policies von the response times of different applications, but also that of the operating systems on their execution time.

Program:

  • Introduction to real-time systems and scheduling of periodic task (definition, characteristic, requirements and challenges)
  • mono-processor scheduling: independent periodic task, heterogenous task (periodic and aperiodic) and dependent tasks (precedence and resources sharing constraints)
  • Schedulability analysis and scheduling algorithms applied to multi-core real-Time systems.
  • Real-Time operating systems
  • static timing analysis and measurement-based timing analysis to estimate the Worst-Case Execution Time (WCET)
  • Classical Embedded Communications networks: Controller Area Network (CAN) and MIL-STD-1553B.
  • New generation network : Avionics Full-Duplex Switched Ethernet (AFDX) and SpaceWire.
Tag: C, Python, Cheddar, PRISE, RTEMS, SimSoWeb

Model Driven Software Engineering

This course deals with the implementation of correct code through a model-based approach to code generation and validation techniques. This course is made up of two main parts:

  1. MBSE (Model-Base Systems Engineering) and synchronous approach for critical software.
  2. Formal methods for software engineering

The first part of the course has the following objectives:

  • Learn to write the systems requirements.
  • Understand the modeling at systems level by using SysML as a support language.
  • Implement the synchronous approach to the model and produce its critical software.
  • Implement a modeling method by starting from the systems and ending at the software.
  • Experiment with the benefits of tools that can detect our design error on the model itself (Model-Checking method)
  • Understand the ins and outs of critical code generation, particularly for the critical applications in human life.

In term of software and tools:

  • The SysML models will be edited and simulated / verified using the free software called TTool from Telecom Paris (TTool implements concepts published in articles co-signed by the IpSC research group of the DISC department of ISAE-SUPAERO and by Telecom Paris LabSoC laboratory).
  • The Ansys SCADE workshop will be used to edit synchronous models, simulate / verify these models, and generate codes that are ready to be embedded on the card.

The second part of the course presents several software validation techniques which is the activity that consists of verifying that a software meets the requirements defining it. This can be done conventionally by testing the software, but for complex and critical systems the formal methods are now used to prove that these systems are correct with respect to their specifications. Formal methods provide mathematical theories and practical tools for reasoning about correcting software.

The aim of this part of the course is to present three different approaches to validate all or part of software:

  • Deductive Methods will be presented either through the C programming language and the ACSL specification language, or through the SPARK language. Demonstrators of automatic theorems like Alt-Ergo, 4 or Z3 will be used.
  • Model-Checking on linear temporal logic (LTL) will be used to prove high-level properties on a program (liveness etc.)
  • Abstract Interpretation is a theory that approximates the behavior of a program and can be used to prove properties on complex software (for example using numerical calculations).

A general introduction to these approaches will be given during the first session of the course. Students will then choose a deepening track corresponding to a particular method (3 or 4 students per method). During the 12 hours dedicated to each "track", students will be supervised by a specialist in each technique and will approach the field from a theoretical but also a practical point of view. A common case study (autopilot, SCAO) will be used and the students will present their work and their conclusions (pros and cons of each technique) during a mini-workshop. Finally, an industrial feedback on the use of these techniques, in particular in the aerospace industry, will be made.

Tag: Ttools, SCADE, The Electrum Analyzer

Distributed Systems

This module addresses the architecture of distributed systems, that is to say for which the processing and data are distributed over several processors or computer units.

Program:

  • Introduction
  • Distributed algorithms
  • Time, order and causality (lobic clock)
  • MPI (Message Passing Interface) and HLA (High Level Architecture )
  • Middleware for distributed systems
  • Transitions systems
  • TLA+
  • Virutualisation and Cloud Computing (Intro)
  • High performance Computing and Big Data (intro)

Tag : C, MPI, OpenMP, TLA+

Avionics Seminars

Embedded systems are tightly integrated with the system governing them to fulfill their mission: controlling an engine, helping the pilot, guiding a vehicle, etc. As such, their hardware and software components must be ready to fulfill their mission, to interact in their environment, but also to resist their environment. In this course, we cover the practical elements to consider when building critical systems for the space and avionics fields.

We first review the generic concepts of IT architectures, hardware and fault tolerance mechanisms. Then, conferences by ISAE industrial partners will return to the main challenges when designing critical systems.

  • Critical Systems
  • On Board Data Processing and On-Board Software
  • Autonomous Systems and Artificial Intelligence
  • Architecture and Design of Embedded Systems
  • Embedded Systems and Avionics Product in Airbus Aircraft

Visiting Lecturers: Seniors Engineers from Aribus and Thales Group.
